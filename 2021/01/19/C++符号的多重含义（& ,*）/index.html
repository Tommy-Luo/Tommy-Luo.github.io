C++符号多重含义（&,*）
Keep team Lv1
2021-01-19 03:51:00
  • C++基础
  • C++

不管是刚刚接触C++的初学者还是已经熟练掌握STL的大佬，引用和指针一直来都是C++或其他类C语言让人头疼的东西，可以说没有真正接触引用和指针，就没有真正意义上学会C++， 而该篇文章主要是就个人观点以及总结《C++Primer》上的相关知识点，对于&和*符号进行个人总结，仅供于个人复习或参考。

对于&

我们都知道，&是取地址符，对于计算机而言，无法识别高级语言，机器本身只能识别01的机械语言，而&相当于是我们能够人为获取计算机内存中的某块地址。
而在C++里，&的用法可不止这一项，还有一种用法，叫做引用。

于是&的含义有：

  1. 作为取地址符，获取相应变量的地址。
  2. 作为引用，相当于是一个已经存在的对象所起的别名。

对于*

如果是刚刚接触编程的朋友或者没写过代码的童鞋，看到“*”想到的肯定是乘法运算，作为类C语言里最令人咬牙切齿的东西莫过于指针了，而在C++里，对于*的用法虽然说可能大部分都是指针的创建和赋值，但是绝对不是全部都是如此。

于是*的含义有：

  1. 作为指针，实现对其他对象的间接访问，其本身就是一个对象，运行赋值和拷贝。
  2. 作为解引用符，访问指针指向的对象

引用的典型案例

int i,&ri = i;
i= 5;
ri =10;
std::cout << i << " " << ri;
//输出结果i = 10,ri =10
return 0;

就如我们刚才所说的，引用本身就是一个已经存在的对象所起的别名，自然而然改变别名的内容，也就改变了原内容，这也就是为什么引用永远都要初始化的原因。

指针的典型案例

int i , *p;
i= 5;
p = &i;
std::cout << i << " " << p << " " << *p;
//输出结果i = 5，p = 0x7ffeefbff4ac , *p = 5
return 0;

这里需要加强记忆了，很多人对于目前而言所认知的指针都是：p存放的是内容的地址，*p存放的是内容本身
当然了，这样的说法是正确的，但是为了我们能够清楚的认识指针，明白*符号的多重含义，我们需要有一个明确的知识记忆。

int i = 5;
int *p = &i;
int *pi = 0;
*p = 0;
std::cout << i << " " << p << " " << *p;

这里将0赋给p所指的内容，也就是修改了p，正如大部分人所说p存放的是指针p的内容，这种访问方法叫做指针的解引用，给解引用的结果赋于新的值，也就是给指针所指的对象赋值。
接下来再看

int i = 5;
int *p = &i;
int *pi = 0;
p = pi;
std::cout << i << " " << p << " " << *p;
//输出结果i= 5，p = NULL ,*p = NUll

这里所做的操作是将空指针pi的内容赋给p，相当于p和pi都指向了空指针。这里不由产生了一个疑问，既然是将pi的内容付给p，那为什么不是p = *p呢？
前面对引用和指针做了简单的个人总结，接下来也就是本篇文章的重点了。

符号的上下文决定了符号的含义。

我们在观察引用和指针的时候发现，引用&ri和指针*p在具体进行赋值操作的时候为什么缺少了之前的&和*符号呢？那是因为他们在定义语句中时，分别写成了&ri和*p的形式：

int &ri = i
int *p = i

其中对应的ri和p是变量名，通过定义指针类型和引用的方法才将声明符写成&ri和*p的形式，所以在赋值语句中：

ri =10; //改变引用内容
p = &i; //改变指针指向

分别没有了&和*

总结
